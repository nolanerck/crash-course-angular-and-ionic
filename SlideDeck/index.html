What is Angular?

  • 1.x is MVC framework for JavaScript
  • 2+ is MVVM framework for JavaScript
  • Command line tools
  • Lives on top of node and npm
  • Dependency Injection
  • Routes
  • 2-way Data Binding
  • Various other things...

What is MVC?

  • Model View Controller
  • Design Pattern
  • Splits your app up into 3 logical sections.
  • Model – where your data and related logic goes.
  • View – the parts of your app you “view” (HTML, CSS).
  • Controller - hooks Model and View together, passes data between them
  • The "restaurant" analogy

What is MVVM?

  • Model View View Model
  • Design Pattern
  • Splits your app up into 2 logical sections.
  • Model – where your data and related logic goes.
  • View – the parts of your app you “view” (HTML, CSS).
  • A restaurant on The Jetsons

The “Model”

  • Short for “data model” (kind of)
  • Where your data and business logic lives
  • Data can be a local JS variable, localStorage, REST API, some other ajax request, etc.
  • The app doesn't care
  • Restaurant: the kitchen

The "View"

  • The part of the app you “view”.
  • HTML, CSS, JavaScript
    • <html>
<body>
this part here is the “view”
</body>
</html>
  • Restaurant: the menu and plates of food

MVVM - All Together

		Model                    View
		-data                    -HTML
		-localStorage	-->      -CSS
		-business logic <--      -Bootstrap
		-etc                     -etc

Angular supports 2-way data binding

Angular 2+ Uses TypeScript

  • Remember trying to "fake" inheritance in vanilla JS?
  • Remember "proto" and "prototype"?
  • Remember how painful that was?
  • And you still couldn't do most other OO things correctly
  • Boo!

Angular 2+ Uses TypeScript

  • Enter TypeScript
  • Let's you write real OO style JavaScript
  • Same (or similar) concepts and keywords you know from other languages
  • Constructors, methods, "this", etc all look more like Java, CFML, C#, etc
  • Data is typed - real ints, arrays, etc
  • Yay!
  • Plus all the normal JavaScript stuff you know...and love?

Angular 2+ Uses TypeScript

  • Uses a "transpiler"
  • Command line tool
  • Converts the .ts files into plain .js that browsers understand
  • Baked into Angular CLI
  • You don't have to install or do anything extra

TypeScript is Object Oriented JavaScript

  • And so by design so is Angular
  • All the code you write is in Classes and Objects
  • But you don't have to do anything beyond some basics to get started
  • Use the fancier stuff when you need to, and when you're ready

How to start using Angular?

  • npm install -g @angular/cli

How to start using Angular?

  • Create a new project
  • ng new HelloWorldApp

How to start using Angular?

  • Serve the application
  • cd HelloWorldApp
  • ng serve
  • ng serve --open

What are <these-things>?

  • Angular Components
  • Building blocks for your app
  • Follow the MVVM design pattern
  • Model - your business logic and data
  • View - stuff you see on the screen

Angular Components

  • Easiest way to add a new component
  • ng generate component MetalBandList
  • ng g c MetalBandList

What's the Catch w/ Angular?

  • New Angular version every 6 months
  • Can be hard to find some details in the docs
  • Officlal docs and demos are a bit terse
  • Not everyone loves CLIs
  • A bit of a learning curve

So, That's Angular

  • That (in an extreme nutshell) is AngularJS.
  • (There are LOTS more features we didn't cover.)
  • Can be used by itself for SPA web apps.
  • Not mobile-specific.
  • Now, on to Ionic...

Ionic

  • Is (essentially) mobile-specific.
  • Sits on top of PhoneGap and Angular.
  • Also uses PhoneGap's “magic”.
  • Gives a Bootstrap-esque set of functionality for hybrid mobile apps.

Ionic

  • Performance “obsessed”.
    • Minimal DOM manipulation
    • Removed 300ms tap delay
    • Etc
  • Lots of other neat features.
  • We'll barely scratch the surface today.

Ionic – Getting Started

  • Need Node.js, Ionic packages, Cordova packages.
  • If building apps locally, need Android and iPhone SDKs (or use PhoneGapBuild).
  • Initial installation can be a bit of a pain.
  • Especially if you don't run bleeding edge OS, dev tools, etc.

Ionic – Getting Started

  • ionic start my-app [blank]
  • Generates all files needed to start an app.
  • “blank” is the template name. Several to pick from, we're starting with a bare-bones app.

  • ionic start my-app tabs

Ionic – Serving Your App

  • ionic serve
  • ionic serve --lab

<ion-*> tags

  • Technically Angular components and services under the hood.
  • The stuff that looks like Angular IS Angular.
  • The ion-* tags are the Ionic part.
  • All the Angular you already know works the exact same way!

What's the catch w/ Ionic?

  • Same as any PhoneGap/Cordova project.
  • Need to install the SDKs for each platform.
  • Takes time, hassles, etc.
  • Android isn't super intuitive
  • iOS requires Xcode, latest OSX, etc

So What's the Deal with Routes?

  • Angular team likes to rebuild the "route" stuff.
  • A lot
  • Often
  • With no regard for backward compatibility
  • So do other 3rd parties
  • Make it stop!
  • First non-MVC sticking point
